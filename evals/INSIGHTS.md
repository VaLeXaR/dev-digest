# evals ‚Äî INSIGHTS

Engineering insights for the DevDigest Claude Code harness evals (`evals/`). Read before touching
the eval engine, cases, or CI wiring. Entries are high-confidence guidance unless a later dated
note supersedes them.

## What Doesn't Work

- 2026-07-15: DeepSeek (`deepseek/deepseek-chat`) does NOT dispatch subagents on the tool tiers (`agentTask`/`workflowTask`) ‚Äî it performs the work inline instead, so `dispatch`-kind cases fail even though the model "did the right thing". Do not default the tool tiers to DeepSeek. `google/gemini-2.5-flash` is the only cheap OpenRouter model verified to actually dispatch (measured in `README.md` ¬ß"Which cheap model ‚Äî verified"). DeepSeek is fine on the content tier only.
- 2026-07-15: `agents/architecture-reviewer` cases use `threshold: 1.0` (100% of practices must pass), calibrated against the default subscription model (`claude-haiku-4-5`). On `google/gemini-2.5-flash` the same case scores ~83% (5/6) ‚Äî the model flagged the right violation but named the rule `no-http-objects-in-services-repositories` instead of the expected `inward-only-dependencies`, so the judge (also Gemini) marked that practice FAIL. Lesson: strict Claude-calibrated thresholds red on non-Anthropic tool-tier models over wording, not substance. To keep the agent tier BLOCKING, either run it on an Anthropic slug via OpenRouter's Skin (`anthropic/claude-haiku-4.5`, no proxy needed) or relax the thresholds; running it on Gemini Flash + threshold 1.0 will flake red (observed CI run 29409404919).
- 2026-07-15: `pnpm vitest run "agents/<name>"` (bare) is a path **substring** filter ‚Äî it also matches sibling dirs sharing the prefix (`architecture-reviewer` pulled in `architecture-reviewer-lite`, running a broken orphan eval in the wrong matrix job). Always anchor with a trailing slash: `"agents/<name>/"`. Fixed in `.github/workflows/harness-evals.yml`.
- 2026-07-15: `evals/agents/architecture-reviewer-lite/` is an ORPHAN eval ‚Äî it calls `runAgentCases("architecture-reviewer-lite", ...)` but no `.claude/agents/architecture-reviewer-lite.md` exists, so every case errors `agent not found`. `ci-detect.mjs` never routes to it (no backing agent file), so post-anchor-fix it is dead in CI; either create the agent or delete the eval dir.

## Codebase Patterns

- 2026-07-15: Split the OpenRouter model by tier in CI, not one model for all. Content tier (`skills`) talks to OpenRouter natively (no proxy) ‚Üí DeepSeek works and is cheapest. Tool tiers (`agents`/`workflow`) run inside the Claude Agent SDK, which speaks the Anthropic wire protocol ‚Üí they need the bundled LiteLLM proxy (`pnpm proxy:up`) + a dispatch-capable model (Gemini Flash). See `.github/workflows/harness-evals.yml` (skills job has no `OPENROUTER_BASE_URL`; agents/workflow jobs point it at `http://localhost:4000`).
- 2026-07-15 (SUPERSEDES the proxy+Gemini part above for the DEFAULT config): tool tiers now default to `anthropic/claude-haiku-4.5` via OpenRouter's Anthropic Skin ‚Äî NO proxy, NO docker. `src/runtime/env.ts:32` defaults `ANTHROPIC_BASE_URL` to `https://openrouter.ai/api` (the Skin) when `OPENROUTER_BASE_URL` is unset, and the Skin serves `anthropic/*` slugs directly. So the agents/workflow jobs dropped `OPENROUTER_BASE_URL` and all `docker compose`/`proxy:wait` steps. Chosen because Anthropic slugs both dispatch AND match the tool-tier cases' Claude-calibrated thresholds (Gemini Flash red-flaked `threshold: 1.0` cases over wording). The proxy path still exists for anyone who sets `EVAL_TOOLS_MODEL` to a non-Anthropic slug ‚Äî they must restore `OPENROUTER_BASE_URL` + the proxy steps.
- 2026-07-15: `scripts/ci-detect.mjs` reads `$CHANGED_FILES` (newline-separated, repo-relative) and maps artifacts ‚Üí suites: `.claude/skills/<name>/**` or `evals/skills/<name>/**` ‚Üí `skills` output; `.claude/agents/<name>.md` or `evals/agents/<name>/**` ‚Üí `agents` output; `CLAUDE.md` / any agent / `evals/src/**` / `evals/workflow/**` ‚Üí `run_workflow=true`. It writes GitHub step outputs directly via `$GITHUB_OUTPUT`.
- 2026-07-15: A changed artifact with NO written evals is reported on the `skipped_skills`/`skipped_agents` outputs ‚Äî deliberately separate from real failures ‚Äî so the PR shows a visible `SKIP <name> (no evals)` instead of going red (`scripts/ci-detect.mjs:56-59`). Preserve this split when editing the detector; conflating them would fail PRs for merely uncovered artifacts.

## Decisions

- 2026-07-15: CI eval gate is wired to the mature `evals/` harness, NOT `skill-evals/` (still a scaffold per its own README) nor the co-located `.claude/skills/*/evals/` data. `ci-detect.mjs` only inspects `evals/skills/<name>/` and `evals/agents/<name>/` for `*.eval.ts`. If the project later adopts the co-located `skill-evals/` runner, the detector and workflow both need rework.
- 2026-07-15: PR gate policy in `.github/workflows/harness-evals.yml`: skills + agents jobs BLOCK merge; the workflow tier is `continue-on-error: true` (non-blocking) because `activation`/`dispatch` cases are flaky on cheap non-Anthropic tool-tier models (rate-limit degradation + behaviour-shaped assertions ‚Äî see `README.md` caveats).
- 2026-07-15: Model is a knob, not code ‚Äî CI resolves `EVAL_MODEL` as `inputs.<tier>_model || vars.EVAL_<TIER>_MODEL || <default>`. Override per-run via `workflow_dispatch` inputs, or repo-wide via Actions variables; never edit the workflow to change models.
